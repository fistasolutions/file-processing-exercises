# Capstone B -- The Team File System

**Build** -- Design a complete file management system for a 3-person team

## Goal

Read `team-scenario.md` describing a 3-person team's file management problems. Design a complete, documented file management system that all three team members can follow independently. This is not a one-time cleanup -- it is a reusable SYSTEM for ongoing file management.

## What You Have

- `team-scenario.md` -- A document describing the team, their roles, current problems, and requirements

## Your Tasks

### Step 1: Understand the Scenario

Read `team-scenario.md` carefully. Note each team member's role, file types, and workflow.

### Step 2: Design Folder Structure

Create a folder hierarchy that accommodates all three workflows without conflict.

### Step 3: Define Naming Conventions

Create naming rules for each file type (reports, designs, code, meetings, etc.).

### Step 4: Write Organization Rules

Define where each type of file goes, with rules for ambiguous cases and a fallback.

### Step 5: Design Backup Strategy

Define what gets backed up, how often, and where.

### Step 6: Create Search Cheat Sheet

Document how each team member can find common files quickly.

### Step 7: Compile Everything

Create `TEAM-FILE-SYSTEM.md` -- a single document comprehensive enough for all three team members to follow independently.

## Expected Results

- A `TEAM-FILE-SYSTEM.md` covering:
  - Folder structure with rationale
  - Naming conventions per file type
  - Organization rules (complete, unambiguous)
  - Backup strategy
  - Search cheat sheet
  - Onboarding guide for new team members

## Time Estimate

2-3 hours for the complete system design.

## Reflection

1. How did you resolve conflicts between team members' workflows?
2. If a 4th team member (a marketing person) joined, what would you add?
3. What is the hardest part of designing a system that multiple people will actually follow?
