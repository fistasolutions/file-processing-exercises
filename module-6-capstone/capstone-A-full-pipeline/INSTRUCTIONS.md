# Capstone A -- The Full Pipeline

**Build** -- Apply the complete 7-step file processing workflow to an 80+ file mess

## Goal

Take the `messy-downloads/` folder through the entire File Processing Framework, from initial survey to final documentation. This is where every skill from Modules 1-5 comes together in a single continuous workflow. You are not just cleaning up -- you are building a reusable pipeline.

## What You Have

- `messy-downloads/` -- 80+ files simulating a real Downloads folder: PDFs, images, ZIPs, code files, spreadsheets, documents with random naming, varied dates, duplicates, empty files, and miscellaneous junk.

## Your Tasks

### Step 1: Survey

Create a complete inventory of all 80+ files. File types, sizes, dates, naming patterns, duplicates, suspicious files.

### Step 2: Backup

Create a timestamped backup of the entire folder. Verify file counts and sizes match.

### Step 3: Plan

Design organization rules and naming conventions. Write them in `pipeline-rules.md`. Rules must handle every file type present and include a fallback.

### Step 4: Test

Apply your rules to one file from each category. Verify placement and naming are correct.

### Step 5: Execute

Organize and rename all files according to your rules.

### Step 6: Verify

Confirm: all files accounted for (count before vs. after), no files in wrong categories, naming is consistent.

### Step 7: Document

Create `PIPELINE-LOG.md` documenting every step: what you did, what you found, decisions you made, and the final state.

## Expected Results

- A fully organized folder structure
- `pipeline-rules.md` -- Reusable organization rules
- `PIPELINE-LOG.md` -- Complete documentation of the process
- Verification proof (file counts match, no files lost)

## Time Estimate

2-4 hours for the complete pipeline.

## Reflection

1. Which step took the longest? Which was most valuable?
2. If you had to run this pipeline again on a different folder, what would you change?
3. What percentage of the work could be automated with a script?
